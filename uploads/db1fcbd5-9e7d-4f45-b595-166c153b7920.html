Platform Overview

BlackVault is a multi-user Capture The Flag (CTF) cybersecurity training platform engineered in the spirit of Hack The Box. It provides a comprehensive hacking simulation environment where participants can sharpen offensive and defensive security skills through structured challenges, hands-on machines, learning paths, and academy modules.

Security Architecture

Authentication Flow

1. User POSTs credentials (email + password) └─▶ Zod schema validation (reject malformed input) 2. Backend calls: bcrypt.compare(password, user.passwordHash) └─▶ 12 salt rounds — resistant to brute-force attacks 3. On success: jwt.sign({ id, username, role }, JWT_SECRET, { expiresIn: '7d' }) └─▶ HS256 signed token 4. Token set as: Set-Cookie: token=...; HttpOnly; SameSite=Lax; Secure └─▶ Inaccessible to JavaScript (XSS-safe) 5. Every subsequent request: cookie sent automatically by browser 6. Middleware verifies: - Signature validity → reject invalid/tampered tokens - User still exists → reject deleted accounts - Status check → reject BANNED / TEMP_BANNED accounts - Maintenance mode → block non-ADMIN during maintenance

Security Measures

MeasureImplementationThreat Mitigated
Password Hashingbcrypt (12 rounds)Database breach
JWT HttpOnly Cookiescookie-parser, SameSite=LaxXSS, CSRF
Input ValidationZod schemas on all POST/PUTInjection attacks
SQL InjectionPrisma parameterized queriesSQL Injection
Rate Limitingexpress-rate-limit on submissionsBrute-force flags
CORS PolicyStrict origin whitelist (FRONTEND_URL)Cross-origin attacks
Role Guardsauthenticate + requireAdmin middlewarePrivilege escalation
Ban SystemBANNED / TEMP_BANNED with auto-liftAbusive users
XSS ProtectionReact JSX escaping (built-in)Stored/reflected XSS

Running with Docker
✅
Recommended ApproachDocker Compose is the fastest way to get the full platform running — no manual database setup required.

Prerequisites

RequirementMinimum Version
Docker Engine24.0+
Docker Compose2.0+
GitAny recent

Quick Start (3 Commands)

BASH 
# 1. Clone and enter project
git clone https://github.com/your-org/ctf-platform.git
cd ctf-platform

# 2. Configure environment
cp backend/.env.example backend/.env
# Edit backend/.env and set JWT_SECRET, DATABASE_URL etc.

# 3. Launch full stack (MySQL, Redis, Backend, Frontend, Nginx)
docker compose up -d

# 4. Seed demo data (optional)
docker compose exec backend npm run db:seed

BASH

sudo sh get-docker.sh

Step 3 — Deploy Application

BASH
git clone https://github.com/your-org/ctf-platform.git /opt/ctf-platform
cd /opt/ctf-platform
cp backend/.env.example backend/.env
nano backend/.env        # Set production values (see Chapter 12)

Step 4 — Systemd Auto-start Service

/etc/systemd/system/ctf-platform.service
[Unit]
Description=CTF Platform
After=docker.service
Requires=docker.service

[Service]
WorkingDirectory=/opt/ctf-platform
ExecStart=/usr/bin/docker compose up
ExecStop=/usr/bin/docker compose down
Restart=always
User=deployer

[Install]
WantedBy=multi-user.target
BASH
sudo systemctl daemon-reload
sudo systemctl enable ctf-platform
sudo systemctl start ctf-platform

Nginx & SSL Setup

Free SSL with Let's Encrypt

BASH
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
sudo certbot renew --dry-run    # Verify auto-renewal works

Production Nginx Config

nginx/conf.d/default.conf
# Redirect HTTP → HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # Security Headers
    add_header Strict-Transport-Security "max-age=31536000";
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    location /       { proxy_pass http://frontend; }
    location /api/   { proxy_pass http://backend;  }
    location /socket.io/ {
        proxy_pass http://backend;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400s;
    }
    location /uploads/ { alias /uploads/; expires 7d; }
}

CI/CD & Terraform

GitHub Actions Deploy Workflow

.github/workflows/deploy.yml
name: Deploy to Production
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /opt/ctf-platform
            git pull origin main
            docker compose up -d --build
